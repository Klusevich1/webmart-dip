# Подробная инструкция по запуску проекта Webmart на другом компьютере

Документ описывает, что нужно установить, какие файлы создать и как запустить монорепозиторий **webmart** (frontend **Next.js 14**, backend **NestJS**, база данных **PostgreSQL**).

---

## 1. Что это за проект

| Часть | Технологии | Порт по умолчанию |
|-------|------------|-------------------|
| **client/** | Next.js 14, React 18, TypeScript, Tailwind | `3000` |
| **server/** | NestJS 10, TypeORM | `3001` |
| **База данных** | PostgreSQL | `5432` |

Корень репозитория — npm **workspaces**: зависимости ставятся одной командой из корня.

---

## 2. Что должно быть на компьютере

Перед запуском проекта проверьте наличие инструментов (если чего-то нет — установите по разделам ниже).

| Инструмент | Зачем нужен | Где описана установка |
|------------|-------------|------------------------|
| **Node.js** и **npm** | Сборка и запуск client/server | [раздел 3](#3-установка-nodejs-и-npm) |
| **Visual Studio Code** | Удобное редактирование кода и встроенный терминал (не обязателен, но рекомендуется) | [раздел 4](#4-установка-visual-studio-code) |
| **Git** | Клонирование репозитория с GitHub/GitLab (если код не копируете архивом) | [раздел 6](#6-получение-кода) |
| **PostgreSQL** и **pgAdmin 4** | База данных и её настройка | [раздел 5](#5-установка-postgresql-и-pgadmin-4) |

**Быстрая проверка в терминале** (PowerShell, cmd или терминал VS Code):

```bash
node -v
npm -v
```

Если команды не найдены — **Node.js не установлен** (или не добавлен в `PATH`), см. раздел 3.

---

## 3. Установка Node.js и npm

**npm** ставится **вместе с Node.js** отдельно качать npm обычно не нужно.

### Windows

1. Откройте сайт **[https://nodejs.org/](https://nodejs.org/)**.
2. Скачайте установщик **LTS** (рекомендуется, версия **18.x или новее**).
3. Запустите `.msi`, пройдите мастер: оставьте включённой опцию **«Automatically install necessary tools»** / добавление в PATH, если установщик предложит.
4. **Закройте и снова откройте** терминал (или перезайдите в систему), затем проверьте:

   ```bash
   node -v
   npm -v
   ```

   Должны отобразиться номера версий без ошибки «не является внутренней или внешней командой».

### macOS

- С **[nodejs.org](https://nodejs.org/)** — так же LTS-установщик **.pkg**, либо  
- через **Homebrew**: `brew install node`

Проверка: `node -v`, `npm -v`.

## 4. Установка Visual Studio Code

Редактор **не обязателен** для запуска проекта (можно использовать любой редактор и обычный терминал), но **VS Code** удобен: подсветка TypeScript/React, встроенный терминал, открытие всей папки проекта.

1. Откройте **[https://code.visualstudio.com/](https://code.visualstudio.com/)**.
2. Скачайте установщик для **Windows** / **macOS** / **Linux** и установите с настройками по умолчанию.
3. Запустите VS Code → **File** → **Open Folder…** → выберите **корневую папку** проекта (где лежит общий `package.json`).
4. Откройте встроенный терминал: **Terminal** → **New Terminal** (или `` Ctrl+` ``). В этом терминале выполняйте команды `npm install`, `npm run dev:server` и т.д. из корня проекта.

**По желанию** — расширения из маркета VS Code: *ESLint*, *Prettier*, *Tailwind CSS IntelliSense* (ускоряют разработку, для первого запуска не обязательны).

## 5. Установка PostgreSQL и pgAdmin 4

1. **Установите PostgreSQL** с [официального сайта](https://www.postgresql.org/download/) под вашу ОС. Запомните порт (по умолчанию **5432**), имя суперпользователя (часто `postgres`) и пароль.

2. **Установите pgAdmin 4**, если он не был включён в установщик PostgreSQL. Без pgAdmin 4 можно обойтись только при работе из командной строки (`psql`, `createdb`); для пошаговой настройки на новом ПК **pgAdmin 4 нужен**. Скачать: [https://www.pgadmin.org/download/](https://www.pgadmin.org/download/)

3. **Запустите pgAdmin 4**, подключитесь к локальному серверу (при первом запуске задаётся мастер-пароль pgAdmin — это не пароль PostgreSQL).

4. **Создайте пустую базу данных** для проекта:
   - в дереве слева: **Servers** → ваш сервер → **Databases** → правый клик → **Create** → **Database…**;
   - **Database**: `webmart`;
   - владелец можно оставить по умолчанию или выбрать пользователя, под которым будет ходить backend;
   - сохраните.

Таблицы приложение создаст само при первом запуске backend (**synchronize: true** в TypeORM — уместно для разработки и демо; в промышленной среде обычно используют миграции).

На Windows при установке PostgreSQL через официальный установщик часто предлагают поставить **pgAdmin 4** в комплекте — согласитесь или установите pgAdmin отдельно по ссылке выше.

## 6. Получение кода

### Если Git ещё не установлен (нужен только для `git clone`)

- **Windows:** установщик с [https://git-scm.com/download/win](https://git-scm.com/download/win), в мастере можно оставить значения по умолчанию. После установки откройте новый терминал и проверьте: `git --version`.
- **macOS:** `xcode-select --install` или установщик с [git-scm.com](https://git-scm.com/download/mac).

Скопируйте папку проекта на новый ПК **или** клонируйте репозиторий:

```bash
git clone https://github.com/Klusevich1/webmart-dip webmart
cd webmart
```

Убедитесь, что вместе с кодом лежат папки `client/`, `server/` и корневой `package.json`.

## 7. Установка зависимостей

Из **корня** проекта (там, где лежит корневой `package.json` с `workspaces`):


Эта команда установит зависимости для **client**, **server** и общие пакеты согласно `package-lock.json`.

## 8. Переменные окружения (обязательно создать файлы)

Файлы `.env` в репозиторий обычно **не коммитятся** — на новом ПК их нужно создать вручную (в VS Code: **New File**, сохранить как `server/.env` и т.д.).

### 8.1. Backend — файл `server/.env`

Создайте файл **`server/.env`** в папке `server`. Пример для **локальной разработки с PostgreSQL**:

```env
# Database (set DB_TYPE=postgres for PostgreSQL, or leave empty for SQLite)
DB_TYPE=postgres
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=root
DB_DATABASE=webmart

# Server
PORT=3001

# Frontend URL for CORS
FRONTEND_URL=http://localhost:3000

# Email settings (configure with your SMTP provider)
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=webmartdip@yandex.by
SMTP_PASS=dvlwgyixxwnhascw
SMTP_FROM=webmartdip@yandex.by

# Agency email (where notifications are sent)
AGENCY_EMAIL=webmart@site.com

# Environment
NODE_ENV=development

GROQ_API_KEY=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
GROQ_MODEL=llama-3.1-8b-instant

```

Подставьте **`DB_USERNAME`** и **`DB_PASSWORD`** того пользователя PostgreSQL, которому разрешён доступ к базе **`webmart`** (созданные в pgAdmin или стандартный `postgres`).

### 8.2. Frontend — файл `client/.env`

Создайте **`client/.env`**:

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

## 9. Заполнение базы начальными данными (seed)

При **`NODE_ENV=development`** при старте сервера выполняется автоматическое заполнение (услуги, блог и т.д.).

Если нужно запустить вручную из корня проекта:

```bash
npm run seed
```

(это вызывает `server`: `ts-node ... src/seed.ts`).

## 10. Локальный запуск в режиме разработки

Сначала убедитесь, что **служба PostgreSQL запущена** (в Windows — в «Службах», в pgAdmin сервер должен подключаться).

Нужны **два терминала**, оба с текущей папкой — **корень репозитория** (во втором терминале VS Code нажмите **+** рядом с вкладкой терминала).

**Терминал 1 — backend:**

```bash
npm run dev:server
```

Ожидаемо: API на `http://localhost:3001`, проверка `http://localhost:3001/health`. В pgAdmin в **Schemas → public → Tables** появятся таблицы после первого успешного старта.

**Терминал 2 — frontend:**

```bash
npm run dev:client
```

Ожидаемо: сайт на `http://localhost:3000`.

Порядок: сначала сервер, затем клиент (чтобы запросы к API не падали при первой загрузке).

## 11. Сборка и production-запуск

Из корня:

```bash
npm run build:server
npm run build:client
```

Запуск:

```bash
npm run start:server
npm run start:client
```

Для production выставьте в `server/.env` как минимум `NODE_ENV=production`, корректные `FRONTEND_URL`, параметры подключения к PostgreSQL и при необходимости `AUTO_SEED=true` **только для первого запуска**, затем отключите, чтобы не перезаписывать данные (уточняйте по политике проекта).

## 12. Полезные команды из корня `package.json`

| Команда | Действие |
|---------|----------|
| `npm run install:all` | То же, что `npm install` |
| `npm run dev:server` | NestJS в watch-режиме |
| `npm run dev:client` | Next.js dev-сервер |
| `npm run build:server` / `build:client` | Сборка |
| `npm run start:server` / `start:client` | Production-старт |
| `npm run seed` | Ручной seed БД |
| `npm run clean` | Удаление `node_modules` (после — снова `npm install`) |

## 13. Проверка, что всё работает

1. В pgAdmin: база `webmart` существует, после старта backend видны таблицы.
2. Откройте `http://localhost:3001/health` — должен ответить успешно.
3. Откройте `http://localhost:3000` — главная страница агентства.
4. Откройте раздел услуг — данные идут с API.

## 14. Типичные проблемы

| Симптом | Что проверить |
|---------|----------------|
| `node` / `npm` не распознаны в терминале | Node.js установлен; терминал **перезапущен** после установки; в крайнем случае перезагрузка ПК |
| CORS / запросы с фронта не доходят | `FRONTEND_URL` на сервере совпадает с URL, с которого открыт сайт |
| `NEXT_PUBLIC_API_URL is undefined` в запросах | Файл `client/.env.local`, перезапуск `npm run dev:client` |
| Backend не подключается к БД | PostgreSQL запущен; в `server/.env` верные `DB_HOST`, `DB_PORT`, `DB_USERNAME`, `DB_PASSWORD`, `DB_DATABASE`; в pgAdmin пользователь имеет доступ к `webmart` |
| Порт 3000/3001 занят | Закройте другие приложения или смените `PORT` / порт Next (`next dev -p 3002`) |
| Пустой каталог услуг | `NODE_ENV=development` при старте сервера или вручную `npm run seed` |

## 15. Безопасность при переносе на другой ПК

- Не копируйте чужие `.env` в публичные места; для коллег создавайте **шаблон** без секретов (например `.env.example` с пустыми значениями).
- Ключи **GROQ** и **SMTP** храните только локально или в секретах CI/сервера.
- Пароль PostgreSQL и доступ к pgAdmin храните вне репозитория.
